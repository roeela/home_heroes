# HomeHeroes — ניהול משימות ביתיות

Flutter + Firebase Android app for a single family to manage home chores.
All UI is in **Hebrew with RTL layout** (Rubik font).

---

## Quick start

### Prerequisites
- Flutter SDK installed
- Android SDK installed
- A Firebase project (see Firebase Setup below)

### 1. Install dependencies
```
flutter pub get
```

### 2. Configure Firebase (one-time)
See **Firebase Setup** section below.

### 3. Run
```
flutter run
```

## Firebase Setup

> Until this is done the app crashes on startup with an `UnsupportedError` from `firebase_options.dart`.

### Step-by-step

1. **Create a Firebase project** at https://console.firebase.google.com

2. **Enable services** in the Firebase Console:
   - Authentication → Sign-in method → Google (enable it)
   - Firestore Database → Create database (start in **test mode** for development)

3. **Install FlutterFire CLI** (once per machine):
   ```
   dart pub global activate flutterfire_cli
   ```

4. **Configure** from inside the `home_heroes` directory:
   ```
   flutterfire configure
   ```
   - Select your Firebase project
   - Choose Android only
   - This overwrites `lib/firebase_options.dart` and `android/app/google-services.json`

5. **Add your debug SHA-1** to Firebase Console → Project Settings → Your Android App → Add fingerprint:
   ```
   keytool -list -v -keystore %USERPROFILE%\.android\debug.keystore -alias androiddebugkey -storepass android -keypass android
   ```
   Copy the `SHA1:` line and paste it into Firebase Console.
   Then **re-download `google-services.json`** from Firebase Console → Project Settings → Your Android App and replace `android/app/google-services.json`.

6. **Deploy Firestore indexes** (requires Firebase CLI):
   ```
   firebase login
   firebase use <your-project-id>
   firebase deploy --only firestore:indexes
   ```
   Alternatively, the app will prompt you with a direct link to create each index the first time a query that needs it runs.

7. Run `flutter run` — the app should launch and Google Sign-In should work.

### Firestore security rules (after dev)

Replace the default permissive rules in Firebase Console → Firestore → Rules:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{db}/documents {
    match /family/{familyId}/{document=**} {
      allow read, write: if request.auth != null
        && exists(/databases/$(db)/documents/family/$(familyId)/users/$(request.auth.token.email));
    }
  }
}
```

## First-time app flow

1. **Primary parent** opens app → signs in with Google
2. No family found → taps "צור משפחה חדשה" → enters family name → family is created
3. Parent goes to **Family tab** → "+" → enters each child's Gmail + display name + weekly quota
4. Each child opens app → signs in with Google → automatically joins the family

## How tasks work

Tasks come in two types:

| Type | Hebrew | Description |
|---|---|---|
| **Weekly Pool** | שבועי | Parent defines once; persists week-to-week. Set how many times per week (1–7). Children register for specific day(s) — each day holds one child, total capped at `availablePerWeek`. |
| **Specific Day** | יום ספציפי | Parent defines for the current week only with specific days. Does not carry over. One child per day, first-come-first-served. |

Each chore appears as **a single card** on the child's home screen regardless of how many slots it has.

### Registration flow
- Child taps a chore card → day-picker bottom sheet opens
- Child taps an available day to register (or taps their own day to unregister)
- Instances are created in Firestore only at registration time — never upfront

### Points & rewards
- Children earn points when a parent approves a completed task.
- Points count toward a **weekly quota** set per child.
- Excess points beyond the quota are shown as "נקודות לפרס" on the child's home screen.
- Child taps "נקודות לפרס" → sends a bonus claim request to the parent.
- Parent sees the request in the **Approvals tab** and taps "תן פרס" to award the real-life reward and zero out the excess.
- Unspent excess (not claimed as a reward) carries forward to the next week automatically.
- Debt (earning below quota) also carries forward as a negative carryover.

### Week
- The Israel week runs **Sunday → Saturday**.
- A new week's balance doc is created automatically when the child first opens the app each week.

### Parent dashboard tabs

| Tab | Label | Contents |
|---|---|---|
| 0 | סקירה | Per-child cards showing weekly progress, excess reward points |
| 1 | משימות | List of all chores; tap to edit, long-press to delete, FAB to create |
| 2 | אישורים | Completed chore approvals + pending bonus claim requests |
| 3 | משפחה | Family members list; primary parent can add/remove members |

### Child dashboard tabs

| Tab | Label | Contents |
|---|---|---|
| 0 | הבית | Weekly goal ring → stat rows → one card per visible chore → day-picker on tap |
| 1 | המשימות שלי | 4 sections: היום / הקרוב / ממתין לאישור / הושלם |
